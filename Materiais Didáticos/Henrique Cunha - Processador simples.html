Processador simples

Para nossa primeira avaliação vamos implementar o processador abaixo.

A figura abaixo mostra um processador extremamente simples que contém um conjunto de registradores de 9 bits, um multiplexador, um somador/subtrator, uma unidade de controle (FSM). A entrada de dados desse sistema é pela entrada de 9 bits DIN. Os dados podem ser carregados por meio de um multiplexador de 9 bits para os vários registradores, tais como R0, R1, ... , R7 e A. O multiplexador também permite que dados possam ser transferidos de um registrador para outro. Os fios de saída do multiplexador são chamados o barramento que permite a troca de dados entre os componentes do processador.

Dessa forma, a adição ou subtração de números com sinal é realizada usando o multiplexador para primeiro carregar um valor no barramento e, em seguida, colocar esse valor no registrador A. Um vez que isso foi feito, um segundo valor é colocado no barramento, o somador/subtrator realiza a operação desejada, e o resultado é carregado no registrador G. Os dados de G podem também ser transferidos para um dos outros registradores caso seja necessário.

O sistema pode executar operações diferentes a cada ciclo de clock, de acordo com o que for determinado pela unidade de controle. Essa unidade faz com que um dado em particular seja colocado no barramento e controla qual dos registradores deve ser carregado com esse dado. Por exemplo, se a unidade de controle mexe nos sinais R0out e Ain, então o multiplexador coloca o conteúdo do registrador R0 no barramento e esse dado será colocado no registrador A no próximo ciclo de clock.

Esse processador, como qualquer outro, executa operações na forma de instruções. Na tabela abaixo são listadas as instruções que o processador deve suportar para esse exercício. A coluna da esquerda mostra o nome da instrução e seus operandos. O significado da sintaxe Rx ← [Ry] é que o conteúdo de Ry será carregado no registrador Rx. A instrução mv (move) permite que o dado seja copiado de um registrador para outro. Para a instrução mvi (move immediate) a expressão Rx ← D indica que a constate de 9 bits D é carregada para o registrador Rx.

Cada instrução pode ser codificada usando um formato de 9 bits IIIXXXYYY, onde III especifica a instrução, XXX é o registrador Rx e YYY é o registrador Ry. Veja que só necessitamos de 2 bits para codificar nossas instruções, mas usaremos 3 bits por que pretendemos adicionar novas instruções no futuro. Dessa forma, iremos assumir os seguintes valores para as instruções:

- mv: 000

- mvi: 001

- add: 010

- sub: 011

Instruções são carregadas a partir da entrada externa DIN e colocadas no registrador de instrução IR. Para a isntrução mvi, o campo YYY não tem significado e o dado #D tem que ser provido na entrada DIN no ciclo de clock seguinte deposi que a instrução mvi seja armazenada no registrador de instrução IR.

Algumas instruções (como a de subtração) levam mais de 1 ciclo de clock para compeltar, isso acontece por que são necessárias várias transferências pelo barramento. A FSM da unidade de controle passa por essas instruções garantindo os sinais de controle corretos em ciclos de clock sucessivos até que a instrução seja concluída. O processador começa executando a instrução na entrada DIN quando o sinal de Run é setado. Quando o processador termina de executa a instrução, ele seta a saida Done. A tabela a seguir insdica quais sinais de controle podem ser setados a cada passo de tempo para que as instruções da tabela 1 sejam implementadas. Perceba que o único sinal de controle setado no passo de tempo zero é o IRin, dessa forma esse passo de tempo não é mostrado na tabela.

- Crie um projeto no quartus

- Gere um arquivo verilog, inclua em seu projeto e use o "esqueleto" de código verilog abaixo

- Use simulação para verificar se seu circuito funciona

module dec3to8(W, En, Y);

input [2:0] W;

input En;

output [0:7] Y;


reg [0:7] Y;

always @(W or En)

begin

if (En == 1)

case (W)

3’b000: Y = 8’b10000000;

3’b001: Y = 8’b01000000;

3’b010: Y = 8’b00100000;

3’b011: Y = 8’b00010000;
